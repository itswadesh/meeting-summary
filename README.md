# AI Meeting Minutes Command Center

An offline-first, AI-powered meeting minutes generator. Record or upload meeting audio (or paste a raw transcript), and the system transcribes it locally, generates a structured summary using a locally-hosted LLM (Llama 3 via Ollama), and produces a downloadable PDF report — all without sending data to any external cloud API.

Built for environments where data privacy and offline operation matter (e.g. internal/intranet use), the entire AI pipeline — speech-to-text and summarization — runs on local hardware.

## Features

- **Live audio recording** directly from the browser (mic capture) or audio file upload
- **Offline speech-to-text** transcription using `faster-whisper`
- **Offline AI summarization** using a local Ollama instance running **Llama 3**, which structures meeting notes into:
  - Executive Summary
  - Problems Identified & Challenges
  - Solutions Proposed & Decisions Made
  - Action Items, Assignments & Deadlines
- **Anti-hallucination guardrail** — if the audio/text doesn't contain an actual meeting (e.g. silence, noise, gibberish), the AI reports that instead of fabricating a summary
- **PDF report generation** with the summary and full raw transcript, available for download
- **Email dispatch** of the generated PDF + summary via SMTP
- **User accounts** — sign up and log in to a personal workspace
- **Persistent summary history per account** — every summary generated by a user is saved to a MySQL database and listed in their dashboard for later retrieval, even after logging out and back in
- **Dark, terminal-styled command-center UI** built with React + Tailwind

## Prerequisites

Before setting up the project, make sure the following are installed on your system:

1. **Python 3.10+**
2. **Node.js 18+** and npm
3. **MySQL Server 8.0+** (running locally or accessible on your network)
4. **Ollama** — for running Llama 3 locally → [https://ollama.com/download](https://ollama.com/download)
5. **FFmpeg** — required by faster-whisper for audio decoding

### Installing Ollama and pulling the Llama 3 model

```bash
# Install Ollama (see https://ollama.com/download for your OS)
# Then pull the Llama 3 model used by this project:
ollama pull llama3

# Verify Ollama is running and serving the model:
ollama list
```

Ollama must be running (`ollama serve`, or it runs automatically after install on most platforms) on `http://localhost:11434` before starting the backend, since `meeting_summariser.py` connects to it directly.

### Installing FFmpeg

- **Windows:** Download from [ffmpeg.org](https://ffmpeg.org/download.html) and add it to your PATH
- **macOS:** `brew install ffmpeg`
- **Linux (Debian/Ubuntu):** `sudo apt install ffmpeg`

## Setup Instructions

### 1. Clone the repository

```bash
git clone <your-repo-url>
cd meeting-summary-master
```

### 2. Set up MySQL

Create the database that the backend will connect to:

```sql
CREATE DATABASE meeting_minutes_db;
```

The backend uses `mysql-connector-python` to connect, and stores:
- User accounts (with hashed passwords)
- Generated meeting summaries, linked to the account that created them

Set your database credentials as environment variables (see `.env.example` if present, or set directly):

```bash
DB_HOST=localhost
DB_USER=your_mysql_user
DB_PASSWORD=your_mysql_password
DB_NAME=meeting_minutes_db
```

> **Note:** Never commit real database credentials or `.env` files to version control. The `.gitignore` in this repo already excludes `.env` and `.env.example`.

### 3. Backend setup (Python / FastAPI)

```bash
cd minutes-pipeline

# Create and activate a virtual environment (recommended)
python -m venv venv
source venv/bin/activate   # On Windows: venv\Scripts\activate

# Install Python dependencies
pip install -r ../requirements.txt

# Run the backend server
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

The backend will be available at `http://127.0.0.1:8000`.

> On first run, `faster-whisper` will download the `tiny` Whisper model weights automatically (requires internet the first time only — after that it's fully offline).

### 4. Frontend setup (React / Vite)

From the project root:

```bash
npm install
npm run dev
```

The frontend will be available at `http://localhost:5173` and is configured to talk to the backend at `http://127.0.0.1:8000`.

### 5. Configure SMTP (optional, for email dispatch)

Update the SMTP configuration constants at the top of `minutes-pipeline/main.py` (`SMTP_SERVER`, `SMTP_PORT`, `SENDER_EMAIL`) to match your mail server if you want the "send summary via email" feature to work.

## Usage

1. Start MySQL, Ollama (`ollama serve`), the backend, and the frontend (in that order).
2. Open the frontend in your browser and **create an account** or **log in**.
3. Set the meeting date, then either:
   - Click **Initialize Recording** to record audio live via your microphone, or
   - **Upload** an existing audio recording, or
   - Paste a raw transcript into the **Text Stream** box
4. Wait for the backend to transcribe (if audio) and summarize the content.
5. View the structured summary, **download the PDF**, or **email it** to a recipient.
6. Past summaries generated under your account appear in the **Historical Session Vault**, and can be revisited at any time — they are persisted in MySQL against your account, not just kept in browser memory.

## Privacy & Offline Operation

This project is designed so that audio transcription and AI summarization never leave the local machine/network:

- Speech-to-text runs locally via `faster-whisper`
- Summarization runs locally via Ollama + Llama 3
- The only network calls made over the public internet are the one-time Whisper/Ollama model downloads — after that, the AI pipeline works fully offline

The MySQL database and SMTP server are expected to be on the same local network/intranet for a fully self-contained deployment.
